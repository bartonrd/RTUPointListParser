README – TestOutput
This folder contains the output files generated by the Copilot AI tool based on the input files located in the Input folder. These outputs are used for comparison and validation against the reference files in the Expected Output folder.

Folder Structure
Example{N}
│
├── Input
│   └── [Source files used for processing]
│
├── Expected Output
│   └── [Reference output files]
│
└── TestOutput
    └── [This folder – AI-generated output files]


Purpose

To store the AI-generated results for the given test case.
To enable comparison between:

TestOutput (AI-generated)
Expected Output (reference standard)




Usage Instructions


Generate Output
Run the Copilot AI tool using the files in the Input folder as source data.


Save Results
Place the generated files in this TestOutput folder. Ensure file names match the corresponding files in Expected Output for easy comparison.


Compare Accuracy
Validate the AI-generated output against the reference files:

Check format consistency.
Verify data correctness.
Document any discrepancies for improvement.




Notes

Do not modify files in this folder manually unless correcting errors for analysis.
Ensure naming conventions remain consistent across all folders.
This folder is specific to Example{N} and should not contain files from other examples.
